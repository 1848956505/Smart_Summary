#### 表格一：控制维度总览

| 类别       | 控制点                       | 对应配置文件参数                                             | 建议概率                    |
| ---------- | ---------------------------- | ------------------------------------------------------------ | --------------------------- |
| 身份与时间 | 用户姓名是否出现             | `NAME_MENTION_PROB = 0.15`                                   | 15%                         |
|            | 时间周期是否明确             | 由 `temporal_pattern` 和 `INCOMPLETE_INFO_PROB` 间接控制     | 见下文                      |
|            | 岗位名称是否明确             | 默认不明确（由模型推断），极少情况可显式给出                 | <10%                        |
| 工作条目   | 日期标注情况（无/部分/全部） | 结合 `temporal_pattern` 和 `INCOMPLETE_INFO_PROB`            | 无日期40%，部分40%，全部20% |
|            | 指标类型（明确量化/模糊/无） | 由 `task_seeds` 内容决定，可配置概率                         | 明确40%，模糊30%，无30%     |
| 风险与待办 | 是否有明确风险项             | 由 `task_seeds` 是否包含风险决定；可配置 `RISK_PROB`（建议新增） | 60%有，40%无                |
|            | 应对策略中是否有时间点       | 由风险描述内容决定                                           | 40%有，60%无                |
|            | 应对策略中是否有负责团队     | 由风险描述内容决定                                           | 40%明确，60%不明确          |
| 下周计划   | 是否有明确计划项             | 由 `task_seeds` 是否包含计划决定；可配置 `PLAN_PROB`（建议新增） | 70%有，30%无                |
|            | 计划期限类型（明确/模糊/无） | 由计划描述内容决定                                           | 明确40%，模糊30%，无30%     |
| 协同支持   | 是否有明确协同事项           | 由 `task_seeds` 是否包含协同决定；可配置 `COLLAB_PROB`（建议新增） | 40%有，60%无                |
|            | 协同方是否明确（@团队）      | 由协同描述内容决定                                           | 明确50%，不明确50%          |
| 全局风格   | 输入格式（散装/列表/段落）   | 由 `SCENARIO_POOL` 和生成指令控制                            | 散装40%，列表40%，段落20%   |
|            | 信息完整度（高/中/低）       | 由 `INCOMPLETE_INFO_PROB = 0.40` 结合各字段缺失情况综合体现  | 高20%，中50%，低30%         |
|            | 噪声类型（错别字、情绪）     | `TYPO_NOISE_PROB=0.35`，`EMOTION_NOISE_PROB=0.45`            | 按概率注入                  |
