# ChatGPT 论文撰写提示词手册

## 1. 文档用途

本文档用于后续向 ChatGPT 提供稳定、可复用、可控的提示词模板，帮助其基于已有系统资料撰写毕业论文正文。其目标不是让模型“自由发挥”，而是让模型在明确边界、统一口径和固定结构下输出更接近论文成稿的内容。

建议使用方式：

1. 先把《系统类论文写作总指导》提供给 ChatGPT。
2. 再把本提示词手册中的对应模板发给 ChatGPT。
3. 最后附上本节所需的系统资料、图表信息和字数要求。

## 2. 使用 ChatGPT 生成论文时的总原则

### 2.1 必须先给模型的基础约束

每次正式生成前，都建议先给 ChatGPT 一段总约束，避免它输出成产品文案、开发文档或空泛套话。

推荐总约束模板如下：

```text
你现在是一名中文本科毕业论文写作助手，请基于我提供的系统资料，撰写“工作总结智能辅助生成系统”论文正文。

写作要求如下：
1. 文风必须是正式、规范、客观的学术论文风格。
2. 内容要体现系统类论文的写法，重点写“需求、设计、实现、测试、效果”，不要写成开发说明书。
3. 不要大段贴代码、接口路径、类名或前后端开发细节。
4. 每一小节都要围绕一个明确问题展开，结构要完整，逻辑要连贯。
5. 要结合系统的真实业务主线：用户、文件夹、周记录、工作碎片、工作总结生成、历史存档。
6. 要突出大语言模型接入系统后的业务价值，而不是只说调用了模型接口。
7. 生成内容应尽量避免空泛套话，必须结合我提供的资料展开。
8. 如果资料中没有的信息，不要擅自编造过细实现细节，可使用“系统采用……方式实现”这类稳妥表达。
```

### 2.2 每次生成都要补充的上下文

除了总约束外，还应根据当前写作任务补充以下信息：

- 当前写的是哪一章、哪一节；
- 这一节需要回答什么问题；
- 可使用的系统资料有哪些；
- 字数范围；
- 是否需要加入图表说明；
- 是否需要衔接上一节或下一节。

## 3. 推荐的资料投喂顺序

为提高输出稳定性，建议按以下顺序投喂：

1. 论文题目与研究对象。
2. 本文档的总约束。
3. 当前章节标题与写作目标。
4. 《系统类论文写作总指导》中的对应小节要求。
5. 系统架构设计文档、数据库设计书、页面截图说明等材料。
6. 明确字数、风格、是否引用图表。

## 4. 通用母提示词

下面是一段最通用的母提示词，后续大部分章节都可以在此基础上改造：

```text
请你根据我提供的材料，撰写本科毕业论文《工作总结智能辅助生成系统》中“{章节标题}”的正文内容。

请严格遵守以下要求：
1. 使用正式、规范、客观的学术论文语言。
2. 内容应符合系统类论文写法，不写成项目开发说明书。
3. 不要出现“我”“我们开发了一个页面”“接口返回了什么 JSON”这类口语化或工程文档式表达。
4. 需要围绕本节核心问题展开，做到逻辑清楚、层次完整、上下文自然衔接。
5. 请结合以下业务主线展开论述：用户 -> 文件夹 -> 周记录 -> 工作碎片 -> 工作总结生成 -> 历史存档。
6. 请突出系统如何通过结构化管理原始记录，并结合大语言模型生成规范化工作总结。
7. 如果涉及系统实现，请重点写实现思路、业务流程和模块作用，不展开代码细节。
8. 如果涉及数据库，请只写到证明设计合理、能支撑业务即可，不要写成字段说明书。
9. 结尾要自然衔接下一小节或下一部分。

本节写作目标：
{写作目标}

本节可参考资料：
{粘贴资料}

字数要求：
{字数要求}
```

2. 再写系统建设目标，即实现碎片记录管理、智能总结生成、历史回看和多用户隔离。

5. 结尾要自然过渡到功能需求分析。

参考资料：
{资料}

字数控制在 600~900 字。

## 8. 用于防止模型跑偏的纠偏提示词

当 ChatGPT 输出开始偏向开发文档、产品宣传或空泛套话时，可使用以下纠偏模板：

```text

1. 当前内容过于像开发说明书/产品介绍，不符合系统类本科毕业论文风格。
2. 请减少代码化、接口化、页面操作说明式表述。
3. 请增强“需求—设计—实现—效果”逻辑。
4. 请结合我提供的系统资料展开，不要空泛套话。
5. 请突出大语言模型接入系统后的业务价值。
```

## 9. 最终成稿前的自检提示词

当某一节初稿已经生成后，可以用下面的提示词做二次检查：

```text
请检查下面这段毕业论文内容是否存在以下问题，并直接给出修改后的版本：
1. 语言是否过于口语化；
2. 逻辑是否松散；
3. 是否偏向开发文档而非论文；
4. 是否存在重复表述；
5. 是否准确围绕本节主题展开；
6. 是否自然衔接前后文。

待检查内容：
{内容}
```

## 10. 推荐的实际使用流程

实际使用时，建议采用下面的节奏：

1. 先用“通用母提示词”生成某一节初稿。
2. 若输出偏短，用“扩写但不跑题”继续完善。
3. 若输出偏散，用“压缩去冗余”重新收束。
4. 若上下节不连贯，用“增强衔接”补过渡。
5. 若风格跑偏，用“纠偏提示词”重新校正。
6. 定稿前，再用“自检提示词”做最后一轮清洗。

## 11. 最后提醒

ChatGPT 适合做“结构化成稿助手”，不适合完全脱离资料自由发挥。为了保证输出质量，必须遵守以下原则：

- 先给规则，再给资料，最后给任务。
- 一次只写一个小节，不要一口气让它写完整章。
- 生成后必须人工检查术语、业务逻辑和是否与真实系统一致。
- 对于资料中没有的实现细节，宁可写得稳一些，也不要让模型虚构。
